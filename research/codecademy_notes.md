## Estrutura curricular complementar identificada em fonte textual aberta

Uma fonte pública textual que replica a sequência do caminho **Data Scientist: Machine Learning Specialist** confirmou uma progressão bastante clara entre fundamentos de dados e especialização em machine learning. A trilha aparece organizada aproximadamente na seguinte ordem: **Principles of Data Literacy**, **Learn SQL**, **Python Fundamentals for Data Science (Part I e II)**, **Python Pandas for Data Science**, **Exploratory Data Analysis in Python**, **Statistics Fundamentals for Data Science**, **Data Visualization Fundamentals with Python**, **Portfolio Project: Data Visualization**, **Data Wrangling, Cleaning, and Tidying**, **Data Science Foundations Portfolio Project**, **Next Step: The Machine Learning Specialty**, **Python Fundamentals Part III**, **Math for Machine Learning**, **Statistics Fundamentals Part II**, **Machine Learning Fundamentals**, **Supervised Learning I**, **Feature Engineering for Data Scientists**, **Unsupervised Learning Algorithms**, **Ensemble Methods in Machine Learning**, **Supervised Learning II**, **Deep Learning and Neural Networks** e **Machine Learning Portfolio Project**.

## Leitura pedagógica para adaptação à trilha da Emily

Essa sequência é especialmente útil porque reforça uma base que conversa bem com uma trilha de Visão Computacional aplicada. Em termos de adaptação, ela sugere cinco blocos que devem aparecer no novo plano da Emily: **fundamentos de dados e programação**, **análise exploratória e visualização**, **matemática e estatística para ML**, **machine learning clássico com engenharia de atributos** e **ponte para deep learning e projetos de portfólio**. Em vez de substituir a trilha de VC, esses blocos devem ser incorporados como uma camada preparatória e de sustentação para PyTorch, CNNs, detecção, segmentação, interpretabilidade, ViTs e engenharia de ML.
