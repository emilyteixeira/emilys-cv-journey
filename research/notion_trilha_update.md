# Trilha Atualizada de Visão Computacional para Emily

## Diagnóstico resumido
Emily já possui base intermediária em Python e experiência prática com OpenCV, PyTorch, CUDA e TensorRT. Por isso, a trilha não deve recomeçar do zero. A atualização reorganiza o plano anterior em uma jornada de Visão Computacional com sustentação explícita em dados, machine learning clássico, estatística aplicada e engenharia de sistemas.

## Princípio pedagógico
A trilha deixa de ser apenas uma sequência de estudos sobre modelos visuais e passa a ser uma formação para construir, avaliar, comparar e operacionalizar sistemas de visão com maturidade de cientista de dados e engenheira de ML.

## Estrutura geral
1. Fase 1 — Literacia de dados, SQL e Python para dados.
2. Fase 2 — EDA, visualização e estatística aplicada.
3. Fase 3 — Machine learning fundamental e feature engineering.
4. Fase 4 — PyTorch reprodutível, treino e CNNs.
5. Fase 5 — Transfer learning e benchmarking controlado.
6. Fase 6 — Detecção, segmentação e interpretabilidade.
7. Fase 7 — Vision Transformers e comparação arquitetural.
8. Fase 8 — ML Engineering, system design e portfólio.

## Integração com a progressão curricular observada
A atualização incorpora a lógica curricular observada no caminho Data Scientist: Machine Learning Specialist, com ênfase em literacia de dados, SQL, Python, EDA, estatística, visualização, machine learning supervisionado e não supervisionado, feature engineering, deep learning e projeto de portfólio.

## Entregáveis centrais
- Fase 1: caderno técnico com pipeline simples de dados e checklist de estudo.
- Fase 2: notebook analítico com visualizações e interpretação estatística.
- Fase 3: benchmark entre modelos clássicos com relatório de avaliação.
- Fase 4: pipeline reprodutível de classificação com análise de erros.
- Fase 5: tabela comparativa entre backbones e estratégias de ajuste fino.
- Fase 6: experimento aplicado com visualização, métricas e Grad-CAM.
- Fase 7: benchmark entre CNNs e ViTs com nota técnica de decisão.
- Fase 8: repositório estruturado, blueprint de arquitetura e caso de portfólio.

## Planner mensal inicial
- Mês 1: dados, SQL e Python para dados.
- Mês 2: EDA, estatística e visualização.
- Mês 3: machine learning fundamental e feature engineering.
- Mês 4: PyTorch, treino e CNNs.

## Planner semanal sugerido
- Domingo: leitura guiada e anotações.
- Segunda: exercício curto ou notebook.
- Terça: reprodução parcial de tutorial oficial.
- Quarta: experimento próprio.
- Quinta: avaliação e análise de erros.
- Sexta: documentação ou benchmark curto.
- Sábado: revisão leve, descanso ou recuperação.
